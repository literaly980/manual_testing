# Тестирование веб-приложения Swag Labs (saucedemo.com)

Учебный проект по тестированию: ручное тестирование интернет-магазина
[Swag Labs](https://www.saucedemo.com/) + автоматизация UI на Playwright.

Объект тестирования — демо-магазин для практики QA: авторизация, каталог товаров,
корзина, оформление заказа. В приложении есть специальные пользователи с
«дефектами» (`problem_user`, `performance_glitch_user`), на которых
продемонстрировано оформление баг-репортов.

## Содержание

| Файл | Что внутри |
|------|------------|
| [`test-cases.md`](test-cases.md) | Чек-лист и детальные тест-кейсы |
| [`bug-reports.md`](bug-reports.md) | Баг-репорты по найденным дефектам |
| [`tests/`](tests) | UI-автотесты на Playwright (Python + pytest) |

## Тестовые данные

Сайт: https://www.saucedemo.com/
Пароль для всех пользователей: `secret_sauce`

| Пользователь | Особенность |
|--------------|-------------|
| `standard_user` | Обычный пользователь (всё работает) |
| `locked_out_user` | Заблокирован |
| `problem_user` | Есть визуальные дефекты |
| `performance_glitch_user` | Медленная загрузка |

## Что проверялось

- Авторизация: позитивные и негативные сценарии, валидация полей.
- Каталог: отображение товаров, сортировка.
- Корзина: добавление и удаление товаров, счётчик.
- Оформление заказа: полный сценарий и валидация формы.

## Запуск автотестов

```bash
pip install -r requirements.txt
playwright install chromium
pytest
```
